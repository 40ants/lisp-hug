I see this same behavior in LWM 5.1.2 64-bit intel, so you are not  
alone. Basically, most keyboard shortcuts, including those defined by  
the user in the Keyboard Shortcuts Systems Pref Pane, requires two  
presses to be recognized by LWM. Sometimes, rarely, it is recognized  
the first time, but it is so much the norm that two key presses are  
required that I've gotten into the habit of hitting cmd-c twice when  
using LWM. Same for cmd-v, cmd-k (which I have bound to Compile and  
Load), cmd-o, etc.

Note that this is not true of editor key chords, including those that  
are defined or redefined by the user. Only command key (aka "apple"  
key) shortcuts.
