Failed to connect, Operation now in progress for socket


I'm running 64 bit LispWorks 5.1.1 on Linux 2.6.15 and am seeing
"Operation now in progress" errors when I open a socket with timeouts
using COMM:OPEN-TCP-STREAM.  This only seems to happen when my machine
starts to saturate its bandwidth.

Is this expected behavior and just part of the timeout logic?

Once the machine gets into this state I get the EINPROGRESS error for
several hours.  I do have a retry loop in my code, but it usually just
fails all the retries.

If memory serves, this is the behavior I would expect from a non
blocking socket if I was calling the C functions directly and the
connection wasn't ready.  It that case I would start a select with
some timeout.
