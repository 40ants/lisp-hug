Belgium (French speaking part) only bug?

Hello,

On Lispworks 5.1.1 running on Windows XP SP2, when the default input language in the 'Languages' tab of the Control Panel 'Regional and languagess options' (Options régionales et linguistiques) is set to 'Français (Belgique)', a grave accent (backquote) is inserted before the first character typed in the listener.

Examples:

If I start a fresh lisp and type '(' in the listener, I get:

CL-USER 1 > `(

If I start a fresh lisp and type 'e' in the listener, I get:

CL-USER 1 > è


This doesn't happen when the default input language is set to 'Anglais (États-Unis)'
This doesn't happen in Lispworks 5.0.0

Regards,
Francis
