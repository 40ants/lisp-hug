Re: LW on Big Sur

Not sure if everybody knows how to deal with problems like that:

If one sees such an error, call in the error REPL

CL-USER 2 : 1 > :bug-form "give the error an descriptive title here"

Then one gets a bunch of output, including fields for more descriptions about the problem, a detailed backtrace, etc.

Add some information to it and send the output to the LispWorks support address, not the mailing list here.


For stuff like that it is likely that there is a patch available... ;-)

Re: LW on Big Sur

Hi,

I confirm that LispWorks 7.1.2 (64-bit) runs "perfectly" on Big Sur, with the Motif-based GUI, so are all command-line LW images. ^_^

P. S. Xquartz 2.7.11 still works on Big Sur. The packages "openmotif" and "imlib2" can be easily installed from Homebrew. I didn't try the GTK-based GUI (but I guess it will also work), because LW only supports the X11-based GTK 2.x installed through Macports.

Regards,

Chun Tian

Re: LW on Big Sur

Yes, this is a known problem with LispWorks on Big Sur.

For the Hobbyist/HobbyistDV/Professional/Enterprise Edition users, we have
just uploaded a fix for it here:

http://www.lispworks.com/downloads/patch-selection.html#lw71bigsur

We will update that page if we find any other problems and make a new build of
the LispWorks Personal Edition.
