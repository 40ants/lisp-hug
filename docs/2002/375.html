stdout, stderr, *standard-output* and *error-output*

In LWL, it looks like *both* *error-output* and *standard-output* go
to stdout by default.  For instance with a file foo.lisp with:

  (format *standard-output* "~&stdout?~%")
  (format *error-output* "~&stderr?~%")
  (quit)

then

  lw -init foo.lisp

will print "stdout?" and "stderr?" to stdout.

It would be nice if this wasn't the case, but it might be hard (maybe
both of them point at *terminal-io*?).

Does anyone know any magic I could do to make them point at the
appropriate Unix streams?

A solution on Windows would be cool, too, though I'm not really sure
what stdout and stderr mean there...
