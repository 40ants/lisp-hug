interface x position

Hi,

This is not a question, just an observation :

When working with more than one screen, the windows of LW don¹t keep trace
of their last positions on the second (or more) screen : they are always
placed on the first one. The y position is correct, but x is set to zero (or
mod by the width of the first screen, I¹m not sure...).
Has I can observe, when creating a interface, it¹s impossible to set it¹s x
position in another screen that the main one by using the :x initiarg.
But the set-top-level-interface-geometry works ok. So, to avoid the problem
a possible solution is to use set-top-level-interface-geometry on the
create-callback of the interface rather than the :x initarg.
I suppose this can be improved in a future release ?
