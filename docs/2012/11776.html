Wild and pathname

Hello,

The Mac file system allows the use of the *¹ character in pathnames, but
lispworks doesn¹t.

(pathname ³/foo*.txt²) works, but
(probe-file (pathname ³/foo*.txt²)) gives an error.
A double backslash before the character seems to resolve the problem :
(wild-pathname-p (pathname ³/foo\\*.txt²)) => nil

Is it a safe solution for a non portable application to insert a double
backslash before any *¹ character in namestrings or pathnames given by the
user ?
