Bug in pathname translations?

Lispworks PE 4.3.6 on WinXP Home.

Create a file containing these two forms:

(setf (logical-pathname-translations "cl-library") 
'((#P"cl-library:**;*.*.*" #P"c:/projects/lisp/cl-library/**/*.*")))
(translate-logical-pathname "cl-library:foo.txt")

Start LispWorks clean and load it. An error occurs in the translate call.

Now change the file so that the setf happens twice, thus:

(setf (logical-pathname-translations "cl-library") 
'((#P"cl-library:**;*.*.*" #P"c:/projects/lisp/cl-library/**/*.*")))
(setf (logical-pathname-translations "cl-library") 
'((#P"cl-library:**;*.*.*" #P"c:/projects/lisp/cl-library/**/*.*")))
(translate-logical-pathname "cl-library:foo.txt")

Restart LispWorks and load the file. The translation works as expected, no 
errors.

Someone please tell me this is a bug. I've been trying to learn how to work 
with the pathname system, and this behavior wasn't helping in the least. If 
it's a feature, please provide a lengthy explanation in simple, short 
English words of why on earth this should behave this way. (BTW, I tried 
leaving the host name out of the from-strings, following some other 
examples I've seen on the net for pathname translations. But I couldn't get 
that to work at all.)

Thanks in advance...
