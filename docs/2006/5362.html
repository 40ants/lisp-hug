The CLOS MOP specification states that the slot-xxx-using-class  
functions are specialized on slot definition metaobjects (among other  
parameters). LispWorks doesn't implement this correctly, but  
specializes on slot names instead.

You may want to consider using the Closer to MOP compatibility in  
which I have fixed this.
