LispWorks 5.0 on Suse Linux?

The LispWorks system requirements for LispWorks 5.0 says "RedHat Linux 
(version 9 or later) or a distribution with kernel version 2.4 or later 
that supports NPTL and glibc 2.3.2 or later".

I understand that to mean tested under RedHat but should work with any 
Linux with a kernel version 2.4 or later and so on, though system 
requirements for LispWorks 4.4 explicitly mentions the Suse distribution 
(amongst others).

Has anyone yet used LispWorks 5.0 on a Suse distribution?  If so, what 
version?  Were there any teething/set-up problems?

I've not used Linux and am not that familiar with it but there is a 
possibility that I might be asked to port a Windows DLL implemented with 
LispWorks 4 to a Suse Linux platform running as some kind of net 
services server.

Thanks for any information you can provide.
