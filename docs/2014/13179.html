Modal dialogs on Windows

Hi,

I have a main interface for an application, and some other interfaces, toolboxes and others. These other interfaces are displayed using the main interface as :owner, making them run in the same process.

Now, if I bring up a dialog with display-dialog, passing the main interface as :owner, I would expect the dialog to block interaction with the main interface as well as the other interfaces, as they are all in the same process. But only the main interface is blocked! The other interfaces are still enabled for interaction from the user.

Is there any way to prevent this? I need an application-modal dialog box, to prevent various race conditions.

(This is not a problem on Mac OS X, only on Windows)
